# 数据广播

::: tip 提示
非 Android 系统可忽略该文档
:::

Android 上可在新数据增加/同步 后发起广播，其他应用可接收该广播进行额外处理(如Tasker)

需要在软件设置里面【剪贴板】-> 【新增数据时发送广播】选项打开

## 广播 Action:

```
top.coclyun.clipshare.ACTION_ON_HISTORY_CHANGED
```

## 广播参数

| 参数名             | 说明      | 备注                                                                 |
|-----------------|---------|--------------------------------------------------------------------|
| `type`          | 内容类型    | 有效值：`text`（文本）、`image`（图片）、`file`（文件）、`sms`（短信）、`notification`（通知） |
| `content`       | 内容      | 当为图片和文件时是本机路径，当为通知时是 JSON 数据                                       |
| `from_dev_id`   | 来源设备 ID | 发送数据的设备唯一标识                                                        |
| `from_dev_name` | 来源设备名称  | 发送数据的设备名称                                                          |

在 Tasker 中可使用 `%` 前缀引用这些参数，例如：`%type`、`%content`
